# Установка на Android &nbsp;·&nbsp; [🇬🇧 EN](ANDROID.md)
Этот гайд поможет вам установить VPN клиент Turnable на Android с использованием Termux и NekoBox. Предпологается, что у вас есть запущенный сервер Turnable с настроенным маршрутом WireGuard, хотя оно в целом применимо к любому протоколу на основе UDP. Пользователи протоколов на основе TCP могут просто пропустить рекомендации по MTU.

## 1. Установите Termux
Во первых, необходимо установить Termux, так как официального приложения Turnable для Android еще нет. **Не используйте версию из Google Play Store**, так как она сильно устарела. Вместо этого загрузите её со страницы [GitHub Releases](https://github.com/termux/termux-app/releases/latest).

При загрузке выберите правильную архитектуру для вашего устройства:
- **arm64** - Большинство современных смартфонов на Android
- **armeabi-v7a** - Старые устройства Android
- **x86** - Некоторые планшеты и эмуляторы

## 2. Установка клиента
### 2.1. Быстрая установка
Запустите скрипт быстрой установки, чтобы загрузить и настроить клиент Turnable:

```bash
curl -sSfL https://raw.githubusercontent.com/TheAirBlow/Turnable/refs/heads/main/scripts/quick-client.sh | bash
```

Рекомендуется сначала создать отдельную директорию:

```bash
mkdir Turnable
cd Turnable
# ... теперь запустите команду curl сверху
```

### 2.2. После установки
Скрипт создает следующие файлы:

| Файл            | Назначение                                 |
|-----------------|--------------------------------------------|
| `turnable`      | Основной исполняемый файл VPN-клиента      |
| `run-client.sh` | Скрипт для запуска клиента с конфигурацией |
| `update.sh`     | Скрипт для обновления на последнюю версию  |

**Использование:**
```bash
# Запустить клиент
./run-client.sh <listen_addr> <config_file> [additional_args...]

# Обновить до последней версии
./update.sh
```

**Параметры:**
- `<listen_addr>` - Адрес для прослушивания сокета, например, `127.0.0.1:PORT` для локального доступа или `0.0.0.0:PORT` для доступа из сети.
- `<config_file>` - Путь к вашей конфигурации, может быть JSON файлом или текстовым файлом с URL конфигурации.

### 2.3. Настройка с URL конфигурации
Если у вас есть URL конфигурации, сохраните его в текстовый файл:

```bash
# ... напрямую через echo в файл
echo "your_config_url_here" > wireguard.txt
# ... или используйте текстовый редактор
nano wireguard.txt
# ... вставьте URL конфигурации, затем нажмите Ctrl+S для сохранения и Ctrl+X для выхода
```

Затем запустите клиент:
```bash
./run-client.sh 127.0.0.1:5080 wireguard.txt
```

### 2.4. Настройка с JSON-файлом
Если у вас есть JSON-файл конфигурации, поместите его в ту же директорию и используйте напрямую:

```bash
./run-client.sh 127.0.0.1:5080 config.json
```

## 3. Установка NekoBox
NekoBox - это клиент прокси для Android, который будет маршрутизировать ваш трафик через клиент Turnable. Установите его из [официального репозитория](https://github.com/MatsuriDayo/NekoBoxForAndroid) или из магазина приложений вашего устройства.

### 3.1. Настройка правил маршрутизации
Удалите все существующие правила маршрутизации и добавьте следующие для блокировки рекламы и защиты от жучков РКН:

| # | Имя                          | Домен/IP                       | Исходящий  | Назначение                                    |
|---|------------------------------|--------------------------------|------------|-----------------------------------------------|
| 1 | Блокировать всю рекламу      | `geosite:category-ads-all`     | Обход      | Предотвращает отслеживание объявлений         |
| 2 | Обход российских доменов     | `geosite:category-ru`          | Обход      | Обход местных ограничений VPN                 |
| 3 | Обход российских IP          | `geoip:ru`                     | Обход      | Обход местных ограничений VPN                 |
| 4 | Блокировать локальный socks5 | Смотрите ниже                  | Блокировка | Предотвращает раскрытие исходящего IP         |
| 5 | Белый список приложений      | Настроить по своему усмотрению | Прокси     | Маршрутизирует выбранные приложения через VPN |
| 6 | Обход по умолчанию           | `0.0.0.0/0, ::/0`              | Обход      | Обход всего остального по умолчанию           |

**Пользовательский JSON для правила #4:**
```json
{
  "inbound": ["mixed-in", "socks-in"],
  "outbound": "block"
}
```

### 3.2. Настройка WireGuard
Если вы вручную создаете конфигурацию WireGuard, убедитесь, что эти поля установлены правильно:

| Поле              | Значение                 | Примечания                                      |
|-------------------|--------------------------|-------------------------------------------------|
| Server            | `127.0.0.1:PORT`         | Используйте PORT из `run-client.sh`             |
| MTU               | `1280`                   | Рекомендуется для туннелирования через Turnable |
| Private Key       | [Ваш приватный ключ]     | Из конфигурации вашего клиента                  |
| Server Public Key | [Публичный ключ сервера] | Из конфигурации сервера                         |
| Network Address   | [Ваша сеть]              | Из конфигурации сервера                         |

Если у вас уже есть совместимая с NekoBox конфигурация WireGuard, вы можете импортировать её как есть, но вы **должны обновить** поля Server и MTU, как показано выше, иначе она либо полностью обойдет Turnable, либо производительность будет значительно снижена.

## 5. Готово!
Ваше устройство Android теперь настроено на маршрутизацию трафика через VPN клиент Turnable. NekoBox будет управлять вашим VPN подключением, используя настроенные надежные правила маршрутизации.
